# Meeting Minute — Project Instructions

## Before writing any code

1. **Confirm first.** Discuss or confirm the approach with me before implementing, even for changes that seem obvious. A quick confirmation saves wrong implementations.

2. **Verify the API exists.** Check the actual API surface (jar, javap, source) — not just memory or docs. A 30-second verification saves hours of broken refactors.

3. **Follow platform best practices.** For Android: Google's official samples and developer.android.com. For iOS: Apple's samples and HIG.

4. **Correctness over throughput.** The right implementation matters more than speed. Always check docs and best practices before writing code. A correct answer a minute later beats a wrong answer now.

5. **Admit uncertainty.** When unsure whether something will work or look right, say so upfront rather than presenting it as fact.
